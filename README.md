# Документация по проекту "Управление проектами и работниками"

## Описание проекта

Проект "Управление проектами и работниками" представляет 
собой REST API, разработанное на платформе ASP.NET Core. 
API предоставляет возможность управления информацией о 
проектах и работниках. 
API позволяет создавать, просматривать, редактировать и 
удалять данные о проектах и работниках, а также 
осуществлять назначение и управление задачами на проектах.

## Запуск проекта

Для запуска проекта, выполните следующие шаги:

1. Убедитесь, что на вашей системе установлен Docker и Docker Compose. Если их нет, установите их согласно официальным инструкциям.

2. Разархивируйте папку с проектом или можете склонировать
репозиторий проекта на свой компьютер:

```shell
   git clone https://github.com/KimIlia91/ProjectManager
```

3. Перейдите в корневую папку проекта и убедитесь что в корневой папке проекта есть файл 
`docker-compose.yaml` 

4. Запустите проект с помощью команды Docker Compose в 
командной строке или через PowerShell:

```shell
   docker-compose up -d
```

5. API будет доступно по адресу 
`http://localhost:8080` (или другому адресу, если вы 
настроили порты в файле `docker-compose.yaml`).

6. Для доступа к Swagger, перейдите по следующему адресу в 
вашем веб-браузере: `http://localhost:8080/swagger`

## Использованные технологии

Проект "Управление проектами и работниками" использует следующие технологии и библиотеки:

- **ASP.NET 7:** Веб-фреймворк нового поколения, 
предоставляющий множество улучшений и новых возможностей для разработки веб-приложений.

- **MS SQL Server:** Реляционная система управления 
базами данных (RDBMS), используется для хранения и
управления данными приложения.

- **Entity Framework Core:** ORM 
(Object-Relational Mapping) для взаимодействия с 
базой данных. Проект использует подход Code First 
DB Migrations для управления схемой базы данных.

- **ASP.NET Identity:** Инфраструктура 
для аутентификации и авторизации пользователей.

- **MediatR:** Библиотека для реализации паттерна 
"Mediator" и управления командами и запросами.

- **Fluent Validation:** Библиотека для декларативной 
валидации данных и входных параметров.

- **Docker:** Проект развертывается в контейнере Docker, 
что упрощает установку и запуск приложения в различных
окружениях.

- **C#:** Язык программирования для разработки серверной 
логики.

- **RESTful API:** Проект следует архитектурному стилю
REST для взаимодействия с клиентами.

- **Разделение ролей:** В проекте реализована система 
разделения прав доступа на основе ролей.

- **Swagger:** Для удобства документации и тестирования 
API проекта, вы можете воспользоваться Swagger.

- **JWT Bearer Token:** Используется для обеспечения 
аутентификации и авторизации пользователей в API.

- **Mapster:** Библиотека для маппинга объектов. 
Mapster облегчает процесс преобразования и сопоставления 
данных между объектами.

## Подходы программирования

Проект "Управление проектами и работниками" стремился 
придерживаться следующих архитектурных стилей и подходов:

- **Модель-Представление-Контроллер (MVC):** Для разделения 
 приложения на компоненты модели, представления и 
 контроллера для удобной организации кода.

- **Использование Entity Framework с подходом Code First:** 
Проект использовал Entity Framework Core с подходом 
Code First DB Migrations, что позволяет управлять 
схемой базы данных с помощью кода.

- **Архитектурный стиль Clean Architecture:** Стремился 
придерживаться архитектурного стиля 
Clean Architecture, который способствует разделению 
бизнес-логики и инфраструктуры, обеспечивая легкость 
сопровождения и расширения приложения.

- **Шаблон CQRS (Command Query Responsibility Segregation):** 
Проект применял шаблон CQRS для разделения операций 
записи (команд) и операций чтения (запросов), 
что позволяет оптимизировать производительность и 
масштабируемость.

- **Шаблон репозитория и базовый репозиторий:** Проект 
использовал шаблон репозитория для управления 
операциями доступа к данным, а базовый репозиторий 
обеспечивал общие функциональности для работы с 
хранилищем данных. 

Преимущества этих подходов включают повышенную читаемость, 
легкость сопровождения и масштабируемость кода, 
а также удобство управления базой данных с 
помощью Code First подхода. 
Несмотря на ограниченное время и опыт, 
использование этих стилей способствовало созданию 
более менее структурированного и гибкого приложения.
